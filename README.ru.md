# RUDP-CLI

Консольная утилита для надежной передачи файлов, построенная поверх протокола UDP. Проект реализует протокол прикладного уровня, обеспечивающий надежность стандартных DatagramSockets.

## Возможности

* **Надежность:** Реализация алгоритма Stop-and-Wait ARQ для обработки потерянных пакетов.
* **Целостность:** Обнаружение поврежденных данных с помощью контрольных сумм CRC32 в заголовке.
* **Хранение:** Автоматическое управление созданием директорий по запросу клиента.
* **Без зависимостей:** Используются только стандартные библиотеки Java.

## Использование

Требуется Java 17+ и скомпилированный jar-файл.

### Запуск сервера

```bash
java -jar rudp.jar server [optional_root_path]

```

### Отправка файла

Файл -> целевой сервер

```bash
java -jar rudp.jar send <source_file> <server_ip> [remote_path]

```

## Реализация протокола

Система обходит TCP, управляя надежностью на уровне пользователя (user-space):

* **Структура пакета:** 13-байтовый бинарный заголовок (Sequence ID, Type, CRC32) + 8 КБ полезной нагрузки (Payload).
* **Handshake:** Трехэтапное согласование имени файла и места назначения перед началом передачи данных.
* **Управление потоком:** Тайм-ауты и повторные отправки гарантируют доставку даже в условиях нестабильного сетевого соединения.
